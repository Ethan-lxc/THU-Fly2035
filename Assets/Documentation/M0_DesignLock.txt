一、操控主规则（定稿）
--------------------------------------------------------------------------------
• 主操控：八向连续向量（键盘 WASD 或左摇杆；斜向为归一化后的对角方向）。
• 与现有 IsoDroneController：M1 起在同一脚本内改为「输入向量 → 目标速度」，
  弃用「鼠标点地 + 轴测路径」作为唯一移动方式（避免两套逻辑并存）。
• 运动手感：带加速度上限与可选 SmoothDamp（与计划 M1「输入向量 → 速度」一致）；
  悬停微动可作为视觉子层保留。
• 相机缩放：首版不强制；与 CameraFollow 扩展参数后续在 M2 统一调参。


二、进度主规则（定稿，三选一已选定）
--------------------------------------------------------------------------------
• 主规则：纯计数 — 已发现兴趣点数量 / 世界内可发现总数（HUD：Found / Total）。
• 不采用：能量返航、区域解锁作为主进度条（可作为 M5+ 扩展线）。


三、失败主规则（定稿，三选一已选定）
--------------------------------------------------------------------------------
• 主规则：无硬性失败 — 无 Game Over、无强制坠毁结算；玩家可随时继续探索。
• 不采用：坠毁返航、信号丢失作为主失败条件（若后续要张力，可作为可选模式另列）。


四、输入方案（定稿：旧 Input Manager，非 Input System）
--------------------------------------------------------------------------------
• 选定：Unity 旧版输入（UnityEngine.Input + Input Manager 轴/键）。
• 理由：
  (1) 当前 Packages/manifest.json 未引入 com.unity.inputsystem，避免 M0 额外依赖与混用；
  (2) 现有代码（IsoDroneController 鼠标、PauseMenuUI Escape、mover.cs 的 GetAxis）已基于旧 Input；
  (3) 若未来需手柄映射或重绑定，再在独立里程碑评估迁移至 Input System。
• 命名固定（与计划 M0 一致）：
  Move      → Horizontal + Vertical 轴（键盘 WASD / 箭头已映射）
  Confirm   → 暂定 KeyCode.E 或 KeyCode.Return（UI 确认，与 MainMenu 对接时再统一）
  Pause     → KeyCode.Escape（与 PauseMenuUI 一致）
  Interact  → 暂定 KeyCode.F（兴趣点交互预留）


五、像素规范（一页）
--------------------------------------------------------------------------------
• 渲染管线：2D + URP（与工程一致）。
• 参考分辨率（Pixel Perfect / 游戏设计基准）：640 × 360（16:9）；允许玩家端整数倍放大，
  窗口非整数倍时以文档说明「可能轻微模糊」或强制最近整数倍（由 M2 与 Pixel Perfect Camera 统一）。
• 参考像素每单位（PPU）：地面/角色统一 16 或 32（二选一在首套 Tile 导入时锁定；推荐 Tile 16px → PPU=16）。
• 精灵导入：Filter Mode = Point (no filter)；压缩：根据平台选用无损或低损，避免二次模糊。
• Tile 基底：16×16 像素格；轴测地块在 Tilemap 中按项目网格对齐。
• 无人机占位高度：约 16～24 像素（以最终精灵为准，与 PPU 一致换算到世界单位）。
• 色数建议：32～64 色主调色板（含明暗），UI 可与场景共用或略增。


六、场景结构（定稿）
--------------------------------------------------------------------------------
• 场景文件均位于 Assets/Scenes/：
  Boot.unity     — 首场景：可选持久化单例、首帧加载 MainMenu（或直连 Game 调试开关）。
  MainMenu.unity — 主菜单：开始游戏 → LoadScene(Game)；退出等。
  Game.unity     — 核心玩法：无人机、Tilemap、相机、兴趣点 MVP、HUD。
• 加载策略：单线程性 LoadScene(Single) — Boot → MainMenu → Game；不设 Addressables 为 M0 前提。
• 与工程内既有场景并存：TitleScene.unity、GameScene.unity 为历史/并行流程；构建顺序中置于 Boot/MainMenu/Game
  之后，迁移期可逐步把逻辑收拢到新三场景，避免重复维护两套入口。
